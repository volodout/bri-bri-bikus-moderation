# apis/ — Эталонные OpenAPI

> Статус: **референс**. Для тебя как студента — опция: можешь опираться, можешь писать своё.

Здесь лежит каноничный взгляд на API NeoMarket — то, к чему стремимся. Snapshot, не живая истина.

## Структура

```
apis/
├── b2b/openapi.yaml              — кабинет продавца
├── moderation/openapi.yaml       — модерация товаров
├── b2c/
│   ├── catalog/openapi.yaml      — витрина и поиск
│   ├── cart/openapi.yaml         — корзина и подписки
│   └── orders/openapi.yaml       — оформление и заказы
└── shared/schemas.yaml           — общие типы данных
```

## Как пользоваться

- Открой нужный `openapi.yaml` рядом с user-flow в [`flows/`](../flows/) — flow задаёт сценарий, openapi показывает контракт.
- Можешь скопировать в свою реализацию или написать заново — главное чтобы твой сервис проходил canonic flow.
- При сдаче квеста соответствие проверяется по [`neomarket-protocols`](../../../neomarket-protocols/) — там живой контракт, который двигают команды.

## Связь с neomarket-protocols

`apis/` здесь — snapshot, замороженный в момент публикации канона. Команды дополняют и меняют **протоколы**, а не канон. Если канон расходится с protocols — ориентируйся на protocols.

Snapshot обновляется на milestone-точках семестра.
